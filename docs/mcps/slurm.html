Slurm

System Management

Job submission and management simulation for High-Performance Computing (HPC) environments. Provides Slurm workload manager functionality through the Model Context Protocol.

1.0.0 Version
4 Actions
Jan 15 Updated
Python Language

Installation Methods

Global Installation

uv pip install scientific-mcps

Individual MCP Installation

uv pip install "scientific-mcps[slurm]"

Direct Server

uv run mcp_slurm

Claude Desktop Configuration

{ "mcpServers": { "slurm": { "command": "uv", "args": ["run", "mcp_slurm"] } } }

Available Actions

  • submit_job
    Submit a job to the Slurm queue with specified parameters
    Parameters: job_script, nodes, tasks, time_limit, partition
  • check_status
    Check the status of submitted jobs
    Parameters: job_id, user
  • cancel_job
    Cancel a running or pending job
    Parameters: job_id, signal
  • get_queue
    Get information about the job queue and partition status
    Parameters: partition, user

Usage Examples

Job Submission and Management

// Submit a parallel job submit_job( job_script: "#!/bin/bash\n#SBATCH --job-name=simulation\n#SBATCH --output=result.out\npython simulation.py", nodes: 4, tasks: 16, time_limit: "02:00:00", partition: "compute" ) // Check job status check_status(job_id: "12345") // View queue information get_queue(partition: "compute")

Workflow Integration

// Submit data processing job submit_job( job_script: "#!/bin/bash\n#SBATCH --job-name=data_analysis\nmcp_pandas process_data.py", nodes: 2, tasks: 8, time_limit: "01:30:00" ) // Monitor until completion while (check_status(job_id)["state"] == "RUNNING") { sleep(30); } // Process results with other MCPs load_csv("processed_results.csv")
